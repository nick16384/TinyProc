Typical asm mnemonics:
[INSTRUCTION] [DESTINATION] [SOURCE] ; [Comments]
e.g.
ADD REG1 0x40 ; Adds 0x40 to register 1
or
ADDR REG1 REG2 ; Adds the value of register 2 to register 1

Instruction set:
[Mnemonic]? [Operand 1] [Operand 2] : [Description] (Supports conditions)
[Mnemonic] [Operand 1] [Operand 2] : [Description] (Does not support conditional execution)

Instructions:
NOP : Do nothing for one cycle

Branching:
JMP [Target address] : Jump to address
B? [Target address] : Branch on condition

Flags:
To be implemented: TST, CLZ, CLOF, CLNG

Arithmetic / Logic instructions:
MOV [Target register], [Source register / Immediate value] : Override the target register value with the source value (Copy source -> Target)
ADD [Target register], [Source register / Immediate value] : Add the source value to the target register and store the result in the target register
SUB [Target register], [Source register / Immediate value] : Subtract the source value from the target register and store the result in the target register
INC [Target register] : Increment the target register by 1 (effectively the same as ADD [Target register], 1)
DEC [Target register] : Decrement the target register by 1 (effectively the same as SUB [Target register], 1)
AND [Target register], [Source register / Immediate value] : Logical bitwise AND between source and target register, store to target register
OR [Target register], [Source register / Immediate value] : Logical bitwise OR between source and target register, store to target register
To be implemented:
MUL, XOR, LS, RS, SRS, ROL, ROR

Memory:
LOAD [Destination register] [Memory address] : Load memory at address to register
LOADR [Destination register] [Memory address from register]
STORE [Source register] [Memory address] : Store the value in the source register at the address
STORR [Source register] [Memory address from register]


Conditional codes:
Conditional codes are appended to the end of an instruction mnemonic and specify, that an instruction should only be executed, when its condition is met.
Example:
B 0x01
is equivalent to
JMP 0x01
and will always jump to address 0x01.
However, if we instead use
BZR 0x01
This instruction will only run, if the flag "ZR" ("zero") is set to 1.
The flag will be set if the result of the previous arithmetic instruction was zero.
In the following example, the BZR instruction will execute:
MOV gp1, 1
SUB gp1, 1
BZR 0x01
While this BZR instruction will not run, because 2 - 1 is not equal to 0:
MOV gp1, 2
SUB gp1, 1
BZR 0x01
Additional note: There are always negated / inverted conditions, e.g. NZ ("not zero") is the inverse of ZR ("zero") and so on.

[Conditional code] - [Description] - [Instruction suffix]
Always execute - <None>
When overflow flag set - OF
When overflow flag not set - NO
When zero flag set - ZR
When zero flag not set - NZ
When negative flag set - NG [not implemented yet]
When negative flag not set - NN [not implemented yet]
