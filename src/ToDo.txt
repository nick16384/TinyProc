Interrupts are just special subroutines (they only differ in the fact that the interrupt flags get set)

When interrupt is caused:
1. push SR to stack
2. "Interrupt enable" flag is cleared (set to 0)
3. "Interrupted" flag is set
4. push PC to stack
5. Extract interrupt vector from instruction operand, get address from vector, jump to vector address
6. ... ISR code runs ...
7. ISR eventually ends with the "iret" instruction
8. Pop PC from stack
9. Pop SR from stack
10. Resume execution at PC

Add new status flags:
Interrupt enable (If 0, won't trigger interrupts except NMIs (Non-maskable interrupts))
Interrupted (Has IRQ or is running ISR, cleared at the end of ISR with iret instruction)
-> If an NMI is triggered while the Interrupted flag is set, a double fault is caused, resulting in a reset
New registers:
Stack pointer (Points to absolute address of uppermost stack element)
